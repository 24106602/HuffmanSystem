# Text Compression and Transmission Simulator Based on Huffman Coding

本项目已从基础哈夫曼编码演示升级为文本压缩与传输模拟器。发送端可将原文的长度、字符频率、哈夫曼码表和编码位串封装成传输包；接收端无需原文或原哈夫曼树，只使用传输包内的码表即可恢复文本。

## 功能

1. 输入文本
2. 统计字符频率
3. 构建哈夫曼树
4. 生成前缀编码表
5. 文本编码（压缩）
6. 二进制编码串译码（还原）
7. 展示字符频率表
8. 展示哈夫曼编码表
9. 以横向树形结构展示哈夫曼树
10. 分析编码前后的位数和压缩率
11. 自动验证译码结果是否与原文本一致
12. 保存和加载独立的 Huffman transmission packet
13. 在无原文、无原哈夫曼树的状态下接收并译码数据包

## 编译

在 VS Code PowerShell 中进入项目目录后运行：

```powershell
& "D:\软件\msys2\mys\ucrt64\bin\g++.exe" main.cpp HuffmanSystem.cpp Utils.cpp -o huffman.exe
```

由于当前 g++ 安装在中文目录中，GCC 可能在链接阶段错误解析自身路径。推荐执行一键编译脚本；脚本会临时映射纯英文盘符、生成可直接运行的静态链接程序，并在编译完成后自动恢复环境：

```powershell
.\build.ps1
```

## 运行

```powershell
.\huffman.exe
```

程序启动后会持续显示交互菜单，可输入文本并分别查看频率表、编码表、哈夫曼树、编码结果、译码结果和压缩率。选择 `8` 可清空当前数据，选择 `0` 退出程序。

## 文件 IO

- 菜单 `9`：从文本文件读取全部内容，保留空格和换行，并重新构建哈夫曼树。
- 菜单 `10`：将当前编码结果保存到文本文件，直接回车时默认保存为 `encoded.txt`。
- 菜单 `11`：将当前译码结果保存到文本文件，直接回车时默认保存为 `decoded.txt`。
- 菜单 `12`：将字符、频率和编码保存为可读表格，直接回车时默认保存为 `code_table.txt`。

编码表中的空格、换行和制表符分别显示为 `[space]`、`[newline]` 和 `[tab]`。文件路径既可以是相对路径，也可以是绝对路径。

## Transmission Packet

- 菜单 `13`：把当前编码结果、原文长度、频率表和编码表保存为传输包，默认文件名为 `packet.txt`。
- 菜单 `14`：加载并校验传输包，检查长度、字符频率、二进制位串和前缀码格式。
- 菜单 `15`：建立反向编码表并独立译码，不依赖当前原文或原来的哈夫曼树。

数据包格式：

```text
===== HUFFMAN PACKET =====
OriginalLength: 10
EncodedLength: 15

[CodeTable]
a 5 1
b 3 01
c 2 00

[EncodedData]
111110101010000
```

特殊字符在数据包中使用 `[space]`、`[newline]`、`[tab]` 和 `[carriage-return]` 表示；其他不可打印字节使用 `0xNN` 表示。接收端会验证频率总和、编码长度、重复码和前缀冲突，格式异常时输出 `Packet format error.`。

程序按字节处理输入文本。ASCII 字符会直接显示；UTF-8 中文文本仍能正确编码、译码和验证，其中多字节内容在频率表和编码表中以十六进制字节展示。

压缩率只统计哈夫曼编码数据位，不包含实际文件格式中的编码表、树结构和文件头开销。
